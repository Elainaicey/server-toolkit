# 贡献指南

感谢改进 Server Toolkit。项目优先考虑可理解、可恢复的高频 VPS 操作，而不是功能数量本身。

## 开始之前

1. 从 `main` 创建短生命周期分支。
2. 保持一次提交聚焦一个问题或一组紧密相关的变化。
3. 不引入 profiles、批量套餐、无人值守确认或旧版兼容层。
4. 修改系统的功能必须提供输入校验、确认、`--dry-run`、备份或明确的恢复路径。

## 本地检查

```bash
bash scripts/check.sh
```

推荐安装 ShellCheck。所有变化至少应通过 Bash 语法、目录格式、单元测试和 CLI 冒烟测试。

## 提交软件条目

普通软件只允许映射一个 Debian/Ubuntu 包。请确认包在项目支持的发行版上存在，并写出能帮助用户判断用途的中文说明。安装、版本检测和单项更新必须使用同一个包映射；不要为了“更完整”附带安装可选组件。

专用安装器只适用于无法用一个系统包正确提供的独立产品，必须使用官方来源、校验系统条件，且不能隐式修改防火墙或其他服务。

## Pull Request

PR 描述应说明：解决的问题、用户可见变化、安全或兼容影响、验证方式。涉及 SSH、防火墙、网络、存储或安装器时，请同时描述失败后的恢复路径。

## 发布变更

版本发布前应执行 `bash scripts/release-check.sh v<版本>`。只有维护者可以创建版本标签；公开发布的标签不得移动或覆盖，修复应使用新的补丁版本。

# Changelog

## 0.1.0

- 将静态项目徽章改为仓库内自托管 SVG，并使用 GitHub 原生工作流状态徽章
- 以 MIT License 发布项目源代码
- 建立青、蓝、紫品牌层级与绿、黄、红语义状态色，重构菜单分组、信息面板和操作区
- 软件中心增加当前版本、仓库候选版本、更新状态、目录统计、更新视图、索引刷新与单项更新
- 增加 `serverctl update ID`、项目更新检查和原子自更新入口
- 增加标签一致性检查、发布归档、SHA-256 校验与 GitHub Release 自动化流程
- 建立全新的 Debian / Ubuntu 基线，不兼容旧版结构
- 使用八个一层式功能中心替代混乱的多层模块菜单
- 建立一次一个软件的目录与安装流程
- Docker、Caddy 作为单项软件接入官方仓库，不附带防火墙等额外配置
- Docker 安装会检测冲突包并单独确认，同时提示容器端口与 UFW 的边界
- 提供系统仪表盘、环境检查、主机名、时区、Swap、时间同步、软件包健康、磁盘分析和安全清理
- 提供网络诊断、监听端口与进程查询、BBR、IP 优先级、防火墙、SSH 向导和登录事件
- 提供 systemd 服务与 Journal 管理、Docker 管理、操作审计、配置快照与单文件恢复
- 扩展进程、用户会话、计划任务、重启状态、目标网络诊断、安全基线、Fail2ban、启动错误与 Timer 检查
- 将 Docker 迁入“应用与容器”子目录，并增加资源、Compose、存储卷、网络和安全清理能力
- 重排主菜单信息架构并升级终端视觉层级、状态说明和危险操作提示
- 修复中文标签按字符格式化导致的仪表盘错位，并加入显示宽度感知布局、资源进度条和统一页面顶栏
- 增加资源压力分析、连接会话、TLS 证书检查和手动配置快照
- 将 README 重写为正式项目主页，补充功能矩阵、命令参考、安全模型、数据路径与架构说明
- 增加 `serverctl uninstall`，支持保留数据卸载或彻底清除项目自身数据
- 删除 profiles、无人值守确认、软件套餐、重复工具入口、自动更新、TCP 调优档位、修复中心和报告生成
- 重写安装器，升级时原子替换运行目录并拒绝危险目标
- 将运行入口迁至 `bin/`，核心与功能代码迁至 `src/`，数据迁至 `config/`
- 根目录安装器缩减为稳定引导器，完整安装逻辑迁至 `scripts/`
- 增加离线结构、目录、CLI、备份和安全边界测试

## 0.3.0

- 收紧安装、卸载、备份和状态目录的路径规范化边界，拒绝尾随斜杠、重复斜杠、控制字符及等价系统顶层目录
- 安装器新增源码归档路径、条目类型、文件数量和展开体积检查，并能识别指向正确目标的悬空命令链接
- SSH 安全向导改为适配 root-only VPS 的“root 仅允许公钥登录”，要求存在 root 公钥并验证最终生效配置和 Include 优先级
- Swap 创建升级为事务式流程；创建、初始化、状态记录、启用或 fstab 写入失败时会安全回滚已创建资源
- 正式发布说明纳入仓库和发布门禁，GitHub Release 直接使用与版本对应的人工维护说明，不再依赖自动生成摘要
- 按单 root VPS 定位移除账户管理、`users` / `user` 命令、Crontab 页面、systemd Timer 管理及 `timer` 命令
- 合并系统仪表盘、健康巡检、环境检查和项目检查的重复路径：保留仪表盘与故障排查，统一使用 `serverctl doctor` 检查运行环境和项目完整性
- 收敛服务、Journal、存储、Docker、应用安装与 SSH 登录分析的重复入口；清理操作回归各自的数据所属中心
- 将系统领域拆分为轻量入口、诊断、菜单、进程、软件包、存储、排障和设置模块，并加入入口体积约束
- 新增按需运行策略测试，禁止项目运行时创建 Cron/Timer、部署 systemd Unit、脱离终端执行后台命令或调用后台化工具
- 应用服务中心新增应用版本、候选版本、更新状态、主进程、重启次数、cgroup 关联监听、固定配置与数据资产视图，并提供 `serverctl app ID`
- Nginx、Caddy、Apache 与 Docker 新增官方只读配置检查；Web 服务 reload 会先验证配置与 systemd 能力，失败时明确阻止重新加载
- 应用健康与数据占用保持完全按需执行，不创建监控进程、Cron 或 Timer；配置资产页不显示可能包含凭据的文件内容
- 网络中心新增单接口下钻、TCP 端点探测、mtr/traceroute 链路路径和套接字压力分析，并提供 `probe`、`trace` 与 `interface` 直接命令
- 将网络中心拆分为概览、内核调优、深度诊断和菜单模块，并把领域根文件收敛为纯加载入口
- 安全中心新增 SSH 登录活动聚合和失败来源处置；只允许选择近期失败清单中的明确 IP，并拒绝阻止当前 SSH 会话来源
- 安全基线扩展到 sshd/sudoers 语法、空密码哈希、关键配置所有权与写权限、ASLR 和近期失败登录
- 新增只读故障快速排查与 `serverctl triage`，集中展示资源、失败服务、近期错误、更新/重启、监听面、Docker 和配置快照
- APT 缓存、Journal、配置快照与 Docker 清理回归各自中心，不再提供重复的聚合清理导航
- Docker 新增独立卷备份中心，支持只读压缩归档、元数据与 SHA-256 校验、运行中占用保护、恢复前安全备份、原卷恢复和单项归档删除
- Docker 容器详情新增重启策略管理，存储页增加卷、未挂载卷和网络统计，并继续明确不自动删除存储卷
- 配置快照新增短备注与人工保护；受保护快照不会进入数量/天数批量清理，也不能直接删除
- 新增项目完整性检查并统一为 `serverctl doctor`，只读检查版本、关键模块、软件目录、命令入口、程序权限、托管路径和安装残留
- 安装元数据和彻底清除流程新增独立 Docker 卷备份路径 `/var/backups/server-toolkit-docker`
- 将软件目录从通用核心层迁入软件功能域，并按查询、展示、写操作和交互页面拆分，消除核心层对专用安装器的反向依赖
- 将端口、地址、进程、服务和托管路径校验从运行时中拆分为独立核心模块，为后续安全、服务与容器重构提供统一输入边界
- 将安全中心拆分为概览、Fail2ban、证书、防火墙、SSH 与菜单模块，将服务中心拆分为概览、Journal、Unit、审计与菜单模块
- 将 Docker 实现拆分为资产概览、Compose、容器操作与菜单模块，并为所有领域入口增加轻量化结构约束
- 新增公网暴露分析，将监听所有地址的 TCP/UDP 端口与进程、systemd 单元、Docker 发布端口和 UFW 精确规则关联展示，并提供 `serverctl exposure`
- 将只读软件包更新页升级为交互式系统软件包健康中心，覆盖安全更新、hold、APT 来源、索引时间、缓存、残留依赖、dpkg/依赖修复及重启需求
- 将存储诊断升级为存储中心，新增分类占用、白名单目录的大文件/目录分析、挂载与 fstab 验证及已删除占用
- 新增 SSH 有效配置、当前连接、公钥资产、主机指纹与统一 SSH 安全中心；Fail2ban Jail 增加有效参数、手动封禁和单 Jail 重载
- 服务详情新增内存、CPU、任务数量、退出结果、Unit/Drop-in、正反依赖、关键启动链和失败诊断；Journal 新增时间、优先级、单元与关键词组合查询
- 本地全文件检查会忽略工作区中已经删除但尚未暂存的路径，同时继续验证所有现存跟踪文件和未跟踪文件
- 软件中心升级为发行版仓库、项目官方 APT、GitHub Release、Git 和官方安装器组成的多来源生命周期中心
- 新增来源浏览、官方 Release 更新检查、单项来源切换、可信重新安装和完整性修复入口，以及 `sources`、`official-updates` 命令
- ripgrep、fd、bat、fzf 可在发行版包与项目官方稳定版之间切换；新增 eza、zoxide、Fastfetch、bottom、dust、duf、hyperfine 与 just
- 官方 Release 安装器支持 amd64/arm64 精确资产、GitHub latest stable 元数据、API SHA-256 digest、压缩包路径检查和安装后版本验证
- 为官方二进制记录仓库、资产、版本、路径和本机 SHA-256；拒绝覆盖未托管文件，并在更新、修复和删除前检查完整性
- 软件目录扩展到 168 项，并将 APT 仓库更新与官方 Release 更新拆分展示，避免混淆不同来源的“最新版本”
- 将 Fail2ban 状态页升级为完整管理中心，支持安装、启停、重启、开机策略、Jail 统计、日志、配置重载与解除误封
- 将 UFW 升级为可交互防火墙管理器，支持启停、重载、日志级别、TCP/UDP、端口范围、来源 IP/CIDR 和编号规则删除
- 应用概览升级为应用服务管理页，可直接控制 Docker、Web、缓存、数据库与已安装的 3x-ui 服务
- Docker 增加 Daemon 服务管理入口；运行中和失败服务清单可直接进入详情与生命周期操作
- 服务生命周期操作增加执行后验证与统一反馈，避免命令成功返回但目标状态未生效
- 时间同步增加启用、禁用、同步状态和实际同步服务管理，系统仪表盘增加基础防护状态
- BBR 增加状态面板和仅针对工具托管配置的安全恢复入口
- 优化信息面板边框、彩色状态菜单项、动态操作提示与安全中心信息架构
- 将 Swap 单向创建升级为带状态所有权记录的生命周期管理，支持即时启停和仅删除工具托管文件
- Journal 增加文件一致性验证、持久化状态、启动记录统计以及按时间或容量清理
- Docker Compose 增加项目上下文识别、服务状态、日志、镜像拉取、`up`、停止、重启和不删除卷的 `down`
- 增加 `serverctl swap`、`journal` 与 `compose` 直接入口
- 进程页增加 PID 下钻、命令与 systemd 归属、线程/FD、打开文件、网络套接字、cgroup、nice 和 TERM/KILL 控制
- 为 PID 1 与工具父子进程建立显式保护边界
- 增加 `serverctl process PID` 直接入口
- 软件安装在执行点刷新 APT 索引，明确安装当前软件源最新候选版本，并增加安装后 dpkg 版本验证
- 官方安装器和 Git 来源改为“已安装、可检查官方更新”，避免在未查询远端时错误显示“已是最新”
- 备份中心增加总占用与完整性状态，支持删除损坏快照、保留最近 N 份和按创建天数批量清理
- 增加 `serverctl backup-delete SNAPSHOT` 与 `backup-cleanup` 直接入口
- UFW 放行规则支持 `443/tcp,80/tcp` 等逗号分隔批量输入、混合协议、端口范围和来源限制
- 新增紧凑型单行输入提示，并应用到主机名、时区、网络目标、TLS、SSH、systemd、PID 与备份操作
- 软件安装、官方仓库配置、终端美化更新与卸载增加失败传递和操作后状态验证，避免误报成功
- 备份复制、清单写入、删除与恢复增加路径和结果校验；SSH、BBR、地址优先级等配置写入加强原子更新与失败回滚

## 0.2.0

- 接入 Starship、Oh My Posh 与 Spaceship Prompt，提供面向实际登录用户的安装、互斥启用、更新和安全卸载
- 提示符使用独立托管配置块，切换时保留其他引擎与用户配置，并明确 Nerd Font 位于 SSH 客户端的边界
- 软件目录扩展到 160 项，新增终端美化、网络、开发、安全、监控、运行时与服务软件
- 增加 Oh My Zsh 官方 Git 安装器，支持目标用户识别、`.zshrc` 备份、更新、可选默认 Shell 与边界明确的卸载
- 软件中心增加软件源可用性检测；仓库未提供的软件会显示明确状态并禁用安装操作
- CI 改为逐个检查每个 Shell 文件，兼容 ShellCheck 0.9/0.11，并为跨模块全局变量与测试桩补充精确说明
- 功能分支仅运行 pull request 检查，避免 push 与 pull_request 重复执行；GitHub checkout 升级到 Node.js 24 运行时
- 将 CI 拆分为 `repository-files`、`shell` 和 `tests` 三项检查，并强制所有仓库文件归类验证
- 增加 UTF-8、LF、末尾换行、尾随空白、Markdown 内部链接、YAML、SVG XML、Git 属性和发布元数据检查
- 软件目录曾由 66 项扩展到 106 项，增加系统工具、网络、安全、监控、数据库客户端和 OCI 容器工具
- 软件中心增加按分类浏览与仅看已安装视图，避免目录扩充后出现过长的平铺列表
- 增加 DNS 解析器与 A/AAAA/CNAME 记录诊断，并提供 `serverctl dns [域名]` 直接命令
- TLS 证书检查增加主机名/IP 匹配验证和剩余有效期分级预警
- 将系统修改与维护动作拆分到 `src/features/system/settings.sh`，降低系统中心单文件复杂度
- 增加综合系统健康巡检，汇总资源、存储、systemd、软件更新、DNS、HTTPS、时间同步、防火墙和 SSH 状态
- 增加 `serverctl health`、`serverctl updates`、`serverctl storage` 与 `serverctl logs SERVICE` 直接命令
- 将软件包状态页扩展为更新版本清单、保留包、dpkg 审计与缓存摘要
- 将磁盘分析扩展为块设备、空间、inode、只读挂载和已删除文件占用诊断
- 备份中心增加完整性校验、当前配置差异、安全删除、快照大小与文件数量展示
- Docker 增加容器健康检查，并将容器操作升级为包含状态、镜像、端口、挂载、日志、资源和生命周期的详情页
- 服务中心增加关键词查找、描述、子状态、主进程、重启次数、状态时间与依赖关系

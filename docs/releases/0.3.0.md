# Server Toolkit 0.3.0

0.3.0 是一次面向日常 VPS 运维的完整能力收口。这个版本不再继续扩张重复入口，而是把系统、网络、安全、服务、软件、应用、容器与恢复能力整理为一致、可验证、按需运行的操作工作台。

## 版本重点

### 更清晰的 root-only 工作流

- 主导航稳定为八个职责明确的功能中心，系统与诊断能力优先于软件和应用。
- 移除账户管理、Crontab、systemd Timer 以及重复的健康检查和清理入口。
- `serverctl doctor` 统一检查运行环境和项目完整性，`serverctl triage` 提供一次性只读故障排查。
- Server Toolkit 不创建后台监控、Cron、Timer 或项目 systemd 服务；所有检查只在用户调用时运行。

### 系统、网络与安全能力深化

- 系统管理覆盖资源压力、进程下钻、APT 健康、更新清单、hold、依赖修复、存储分析、Swap 和时间同步。
- 网络中心新增单接口详情、TCP 端点探测、链路路径、套接字压力和更完整的监听面分析。
- 安全中心新增公网暴露关联、SSH 登录活动、失败来源处置、Fail2ban Jail 操作和完整 UFW 生命周期管理。
- SSH 安全向导适配单 root VPS：支持修改端口、禁用密码认证和限制 root 仅使用公钥登录，但不会提供会直接禁止唯一管理员登录的选项。

### 多来源软件生命周期

- 软件目录扩展到 168 个单项条目，继续坚持一次只安装、更新或移除一个软件。
- 支持 Debian/Ubuntu 仓库、项目官方 APT、官方 GitHub Release、官方 Git 与官方安装器。
- ripgrep、fd、bat、fzf、eza、zoxide、Fastfetch、bottom、dust、duf、hyperfine 和 just 可使用经过架构匹配与 SHA-256 校验的官方稳定 Release。
- 增加来源浏览、官方版本检查、来源切换、完整性检查和可信修复；不会覆盖来源不明的 `/usr/local/bin` 文件。

### 应用、服务与 Docker

- systemd 服务详情增加资源、退出结果、依赖、启动链、失败诊断和经过结果验证的生命周期操作。
- 应用中心识别 Docker、Nginx、Caddy、Apache、Redis、Memcached、PostgreSQL、MariaDB 与 3x-ui，并展示版本、进程、监听、配置和数据资产。
- Web 服务 reload 会先调用应用官方配置检查，再验证 systemd reload 能力。
- Docker 增加 Compose 项目管理、容器详情与重启策略、安全清理，以及带元数据和 SHA-256 的卷备份、验证、恢复和删除。

### 恢复与安全边界

- 配置快照支持备注、保护、完整性校验、差异、单项恢复、按数量或天数清理。
- 安装器采用同目录暂存和原子替换，升级失败时恢复旧安装。
- 0.3.0 进一步拒绝尾随斜杠、重复斜杠和等价系统顶层目录，降低自定义安装或数据路径误配置的风险。
- 源码归档在解压前检查路径、条目类型、文件数量与展开体积。
- Swap 创建在中间步骤失败时会停用并清理本次创建的文件、状态和 fstab 变更。

## 兼容性与变更

- 支持 Debian 11、12、13 与 Ubuntu 22.04 LTS、24.04 LTS，要求 systemd；目标架构为 amd64 和 arm64。
- 本版本删除 `health`、`users`、`user` 与 `timer` 命令，以及账户、Crontab、Timer 和重复清理菜单。
- 环境与项目检查统一使用 `serverctl doctor`；综合只读排查使用 `serverctl triage`。
- 3x-ui 只提供已安装服务的状态、日志和生命周期管理。项目不会猜测第三方安装地址或自动升级 3x-ui。
- 软件、系统配置和业务数据不会在项目升级时被自动迁移、删除或批量更新。

## 升级

已安装用户可以运行：

```bash
serverctl self-update
```

也可以重新执行原子安装流程：

```bash
bash <(curl -fsSL https://raw.githubusercontent.com/Elainaicey/server-toolkit/refs/tags/v0.3.0/install.sh)
```

全新安装完成后建议依次运行：

```bash
serverctl version
serverctl doctor
serverctl status
```

## 发布验证

发布归档会经过全文件分类与格式验证、逐文件 Bash 语法和 ShellCheck、离线单元测试、CLI 冒烟测试、按需运行策略检查以及标签、版本、许可证和发布说明一致性检查。真实 VPS 上涉及 SSH、防火墙、Swap、systemd 和 Docker 的写操作仍应先保留服务商控制台与实例快照，再按实际环境逐项验证。
